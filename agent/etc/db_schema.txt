DB name: agentserver

table "machine":
+-------+------------------+------+-----+---------+-------+
| Field | Type             | Null | Key | Default | Extra |
+-------+------------------+------+-----+---------+-------+
| ip    | varchar(16)      | NO   | PRI | 0.0.0.0 |       |
| v     | int(10) unsigned | NO   | MUL | NULL    |       |
| done  | int(1)           | NO   |     | 1       |       |
+-------+------------------+------+-----+---------+-------+

table "version":
+-------+------------------+------+-----+---------+----------------+
| Field | Type             | Null | Key | Default | Extra          |
+-------+------------------+------+-----+---------+----------------+
| v     | int(10) unsigned | NO   | PRI | NULL    | auto_increment |
| files | varchar(2048)    | NO   |     |         |                |
| time  | datetime         | NO   |     | NULL    |                |
+-------+------------------+------+-----+---------+----------------+

提交更新流程（拟）：
一、 在version表中插入新记录，得到了auto_increment的版本号v；
    将要更新的文件上传到：更新“服务器根目录/up/版本号v/<子目录/文件名(.扩展名)>”的位置。
    收集所有上传的<子目录/文件名(.扩展名)>，以分号分隔，存放于version表的files字段，同时将当前时间存入time字段。
    填写配置文件，提供按钮进行配置文件检查。
二、填写要更新agent的IP列表；如果machine表中不存在某IP，则新建一条以IP为主键的记录，该记录的v字段指向本次更新的版本号v，done值置为0；
    如果IP已存在，则更新其v字段指向本次更新的版本号v，done值置为0。
三、agent发来更新请求时，依据请求中的ip，查询machine表，如果done=1则表示已更新过该版本，直接返回空列表；done=0且v对应的files字段有值时，
    返回“版本v;files字段的值”，完毕后done置为1。
